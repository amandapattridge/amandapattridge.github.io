Accessing .env variables with dotenv

14 May 2016

External APIs often require an app to have a password, token or other authorization code in order to send requests to its servers. Publishing these secret codes to GitHub or your deployment server is generally a bad idea; it allows anyone to access them and abuse your access privileges.

To prevent this, we usually store this information in a text file named .env in the project's root directory. This file is also added to .gitignore, so it will not be tracked by git. Similarly, these codes can be added manually to the deployment server, usually on the deployment service's site.

But if our passwords and tokens are stored in a text file, how can we access them in our app?

dotenv is a simple to use module, which allows us to do just that.

First, set up your .env file in your project's root directory

//.env
secretId=john123
secretPassword=1234567

Next, install and save dotenv to your package.json

//terminal    
npm install --save dotenv

In your server file, require dotenv load your .env variables

//server    
var dotenv = require('dotenv');
dotenv.load()

You can now access you .env variables using process.env and use them however you wish.

//server
{
  secretId: process.env.secretId,
  secretPasswod: process.env.secretPassword
}
